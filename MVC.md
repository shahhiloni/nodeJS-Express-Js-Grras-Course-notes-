MVC  pattern 
M - Model 
V - view 
C - controller

Model : Provides logic and validation in Model or as a Schema

View : Provides UI layer (HTML, CSS, ReactJS)

Controller : handle Req and res from routes, connect with models, 


// work flow of MVC pattern 

Step : 1 : User send Request in Backend 

step : 2 : then Request goes into Routes 

Step : 3 : Controller

Step : 4 : check Validation in Model ,set data according to data - types and  store data in database .

Step : 5 : again in controller 

Step : 6 : Represent the UI Component (VIEW)


// Advantages of MVC Pattern 

1. Provide Clean Road -map for Backend 

2. error - finding process is easily (debuging process)

3. support large - scale Project
